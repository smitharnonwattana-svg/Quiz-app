2. DATA SCHEMAS
---------------

attempts (s.attempts: Attempt[]) — written at index.html:2785 (normal exam)
                                   and index.html:4339 (practice):

  {
    id:           string,
    examId:       string,
    examTitle:    string,
    examSubject:  string,                 // e.g. "คณิตศาสตร์", "ไม่ระบุ"
    takerName:    string,                 // student name = userId
    startedAt:    string (ISO),
    submittedAt:  string (ISO),
    usedSeconds:  number,
    score:        number,                 // correct count
    total:        number,                 // total questions
    answers:      { [qid]: chosen },      // normal exam only
    perQuestion:  [{
      qid?, no, page?, tags, topic,
      chosen, correct, isCorrect, isFree, unsure
    }],
    mode?:        'weakness_practice'     // present only for practice attempts
  }

Practice attempts share the same array but include mode:'weakness_practice'.


weakness items (WeaknessTracker, index.html:5591)
- Storage: per-user localStorage key  nanont:weaknesses_{sanitizedUserId}
           plus Firestore doc         weaknesses_{sanitizedUserId}.
- Shape: object map keyed by examId__no, values:

  {
    userId:         string,                  // takerName
    examId:         string,
    examTitle:      string,
    subject:        string,                  // examSubject
    questionNo:     number,
    status:         'active' | 'resolved',
    lastResult:     'wrong' | 'correct',
    wrongCount:     number,
    correctCount:   number,
    firstWrongAt:   ISO string,
    lastAttemptAt:  ISO string,
    resolvedAt:     ISO string | null,
    updatedAt:      ISO string
  }


practiceLogs
- Not a separate array. Practice sessions are stored in s.attempts with
  mode === 'weakness_practice'.
- Stage 1 helper getTodayPracticeLogs(allAttempts, userId) (line 5780)
  filters today's submitted practice attempts.


users / members (s.members: Member[]) — default seed at line 1675/2102:

  { pin: string, name: string }

- Default member { pin: '311257', name: 'ณนนท์' } if list is empty.
- Admin is NOT stored as a member — gated by the ADMIN_PIN constant.

6. NAVIGATION ENTRY POINTS
--------------------------

- All page transitions go through the global navigate(page, params)
  (line 2257). Buttons/cards use inline onclick="navigate('...')".
- Auth-related: Auth.logout() calls navigate('login'). Successful login
  flow elsewhere routes to home.

From page-home (rendered by renderHome):
  - navigate('exams')           — exam list
  - navigate('stats')           — student stats / results
  - navigate('entrance_stats')  — entrance exam statistics
  - navigate('admin')           — admin panel (teacher only)
  - Auth.logout()               — back to login

- No deep links from home into individual exams, weakness practice,
  or coach metrics.
